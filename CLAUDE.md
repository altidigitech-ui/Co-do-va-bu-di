# Co-do-va-bu-di

## Projet
Repo opérationnel distribution-first pour FoundryTwo.
Deux personnes full-time : R (Romain, growth) et F (Fabrice, CTO/builder).
Ce repo gère UNIQUEMENT la distribution. Le build se fait dans des projets Claude séparés.

## Méthode
COMMUNAUTÉ -> DOULEUR -> VALIDATION -> BUILD -> DISTRIBUTION (VOLUME x CONSTANCE) -> REPEAT

## Principes
1. La complexité = le moat. Ne JAMAIS filtrer par complexité technique.
2. Distribution-first. Jamais de build sans validation communauté (10+ signups en 48h).
3. Volume x Constance non-négociable. R : 30 interactions/jour. F : 15/jour.
4. Problèmes à 10K$/an minimum. Le pricing reflète la douleur, pas la feature.
5. Cible NON-dev uniquement. Jamais de produit pour des codeurs.

## Quand R ou F ouvre ce repo
1. Regarder la date du jour
2. Calculer le jour du plan (J1 = 06/04/2026)
3. Ouvrir romain/ ou fabrice/ selon qui parle
4. Lire le plan-30-jours.md + le tracking de la veille
5. Dire exactement quoi faire MAINTENANT
